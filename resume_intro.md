# spring-watch | 项目介绍

**spring-watch** —— 面向 Spring Boot 应用的轻量级一站式监控平台,基于"主动拉取"模型构建。目标是通过单一接入点(Java Agent)采集方法级指标、应用日志与心跳,提供实时告警、聚合分析与可视化能力,帮助运维 / SRE / 研发团队在零业务侵入的前提下完成全链路可观测。

## 核心职责

- **主导架构设计**:定义"拉取模型 + 注解驱动 + 业务零侵入"五大硬约束,产出项目白皮书作为最高级定性文件
- **设计完整数据链路**:目标应用经 Java Agent 暴露指标 / 日志端点,平台 15s 周期主动 HTTP 拉取,经消息队列异步解耦后批量写入时序库
- **自研日志拉取方案**:在 OTel 无原生 pull 端点的能力缺口下,设计内存日志缓冲 + 拉取式 Controller,实现增量日志拉取
- **构建告警引擎**:完整状态机(`IDLE→PENDING→FIRING→RESOLVED`) + JEXL 表达式 + 次数 / 持续时间阈值,支持 SMTP 邮件通知
- **实现日志 Ingest 管道**:解析 → 脱敏 → 指纹去重 → 时序入库,支撑错误率突增、TopN 聚合、日志-指标关联等分析场景
- **稳定性设计**:Per-Host 并发限流 + 拉取重试队列,保证大规模应用并发拉取下的平台稳定
- **规划演进路线**:设计自研 Agent 目标态,接入成本从"4 步 + 1 个 annotation jar"压缩到"1 个 `-javaagent` 参数 + 0 业务改动"

## 项目状态

预览版核心链路已贯通(指标采集 / 日志分析 / 告警 / 可视化 / 自监控),正在迭代正式版。
