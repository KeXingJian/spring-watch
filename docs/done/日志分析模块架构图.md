## 2. 类级别静态结构(类图)

```mermaid
classDiagram
    class LogAggregator {
        -InfluxDBClient influxDBClient
        -String logBucket
        -String influxOrg
        +errorRate(appid, from, to) ErrorRateStats
        +topPatterns(appid, from, to, topN) List~PatternStats~
        +errorRateSeries(appid, from, to, every) List~ErrorRatePoint~
    }

    class ErrorRateStats {
        <<record>>
        +long total
        +long error
        +long warn
        +double errorRate
    }

    class PatternStats {
        <<record>>
        +String fingerprint
        +String pattern
        +long count
    }

    class ErrorRatePoint {
        <<record>>
        +Instant time
        +String level
        +long count
    }

    class LogAnomalyDetector {
        -StringRedisTemplate redis
        -long rateTtlSeconds
        -long patternTtlHours
        -double minBaseRate
        +isErrorRateSpiking(appid, currentRate, multiplier) boolean
        +isNewPattern(appid, fingerprint) boolean
    }

    class LogMetricsLinker {
        -LogAggregator aggregator
        -InfluxDBClient influxDBClient
        -String metricsBucket
        -String defaultMetric
        +correlate(appid, from, to, metric) CorrelationReport
        -queryMetricMean(...) double
        -normalize(v) double
    }

    class CorrelationReport {
        <<record>>
        +long appid
        +String metricName
        +double errorRate
        +double metricMean
        +double score
    }

    class LogAlertScheduler {
        -LogAggregator aggregator
        -AlertRuleRepository ruleRepository
        -AsyncAlertExecutor alertExecutor
        -long windowSeconds
        +evaluateErrorRateRules() @Scheduled fixedRate
    }

    LogAggregator ..> ErrorRateStats
    LogAggregator ..> PatternStats
    LogAggregator ..> ErrorRatePoint
    LogMetricsLinker --> LogAggregator : 调用 errorRate
    LogMetricsLinker ..> CorrelationReport
    LogAlertScheduler --> LogAggregator : 拉取错误率
```

## 3. 核心调用链路(时序图)

### 3.1 错误率查询链路

```mermaid
sequenceDiagram
    autonumber
    participant C as LogController
    participant AG as LogAggregator
    participant IF as InfluxDB(Flux)
    participant R as Redis

    C->>AG: errorRate(appid, from, to)
    AG->>IF: range/filter(group:level)/count
    IF-->>AG: FluxTable
    AG-->>C: ErrorRateStats(total/error/warn/rate)

    C->>AG: errorRateSeries(appid, from, to, every)
    AG->>IF: aggregateWindow(every)
    IF-->>AG: FluxTable
    AG-->>C: List~ErrorRatePoint~
```

### 3.2 异常突增检测链路

```mermaid
sequenceDiagram
    autonumber
    participant C as LogController
    participant AG as LogAggregator
    participant AD as LogAnomalyDetector
    participant R as Redis

    C->>AG: errorRate(appid, now-300s, now)
    AG-->>C: ErrorRateStats
    C->>AD: isErrorRateSpiking(appid, rate, 3.0)
    AD->>R: GET log:errorRate:{appid}
    R-->>AD: lastRate
    AD->>R: SET log:errorRate:{appid} (TTL 600s)
    AD-->>C: boolean spiking
    C-->>C: AnomalyReport
```

### 3.3 TopN 异常模式 + 新模式发现

```mermaid
sequenceDiagram
    autonumber
    participant C as LogController
    participant AG as LogAggregator
    participant AD as LogAnomalyDetector
    participant IF as InfluxDB
    participant R as Redis

    C->>AG: topPatterns(appid, from, to, 10)
    AG->>IF: filter level=ERROR / group(fingerprint) / sort / limit
    IF-->>AG: 聚合结果
    AG-->>C: List~PatternStats~
    loop 每条 fingerprint
        C->>AD: isNewPattern(appid, fp)
        AD->>R: SADD log:knownPatterns:{appid} fp
        R-->>AD: added count
        AD->>R: EXPIRE 168h
        AD-->>C: boolean isNew
    end
```

### 3.4 日志×指标关联打分

```mermaid
sequenceDiagram
    autonumber
    participant C as LogController
    participant ML as LogMetricsLinker
    participant AG as LogAggregator
    participant IF as InfluxDB

    C->>ML: correlate(appid, from, to, metric)
    ML->>AG: errorRate(appid, from, to)
    AG-->>ML: ErrorRateStats
    ML->>IF: range/filter(metric)/mean
    IF-->>ML: metricMean
    ML->>ML: score = errorRate*0.6 + normalize(metricMean)*0.4
    ML-->>C: CorrelationReport
```

### 3.5 错误率告警调度(定时投递给告警引擎)

```mermaid
sequenceDiagram
    autonumber
    participant SCH as LogAlertScheduler
    participant RR as AlertRuleRepository
    participant AG as LogAggregator
    participant AEX as AsyncAlertExecutor
    participant AE as AlertEngine

    Note over SCH: @Scheduled fixedRate=60s
    SCH->>RR: findByRuleTypeAndStatus("log_error_rate","enabled")
    RR-->>SCH: List~AlertRule~
    loop 每条规则
        SCH->>AG: errorRate(appid, now-window, now)
        AG-->>SCH: ErrorRateStats
        SCH->>SCH: 合成 MetricEvent(metric=log_error_rate, value=rate*100)
        SCH->>AEX: submit(synthetic)
        AEX->>AE: 异步执行状态机
    end
```

## 5. 关键设计要点

| 维度 | 设计 | 落点 |
|------|------|------|
| 查询引擎 | InfluxDB Flux + group/count/sort/limit | `LogAggregator` |
| 状态持久化 | Redis TTL 控制窗口记忆 | `LogAnomalyDetector` |
| 关联算法 | 线性加权 score = err*0.6 + norm(metric)*0.4 | `LogMetricsLinker#correlate` |
| 调度策略 | `fixedRate` 而非 `fixedDelay`,避免执行慢导致窗口间隙 | `LogAlertScheduler` |
| 告警对接 | 错误率转 `MetricEvent` 复用指标告警通道 | `LogAlertScheduler#submit` |
| API 边界 | 只读聚合/异常/关联,落库由 ingest/consumer 负责 | `LogController` |
