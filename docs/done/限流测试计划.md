# 限流(降级重投)测试计划

> 目的:验证 `HostThrottler` 触发限流后,`PullRetryQueue` 降级重投能落地,不被"等下个周期"饿死
> 测试环境:本机 Docker Desktop
> 关键参数:`spring-watch.collector.per-host-concurrent=1`

---

## 一、测试目标

| # | 目标 | 验证点 |
|---|---|---|
| T1 | 多 app 指向同一 host,触发限流 | 看到 `拉取被限流` 日志 |
| T2 | **心跳永远先发**(限流不丢活性信号) | 限流日志前面有 `发送心跳` |
| T3 | 限流后入重投队列 | 看到 `入重投队列 - attempt=0` |
| T4 | drainer 抢到 permit,执行重活 | 看到 `重投执行 - attempt=N` |
| T5 | 退避机制生效 | 看到 `重投仍被限流 - attempt=N->N+1, backoffMs=...` |
| T6 | 5 次耗尽会放弃 | 看到 `重投耗尽 - attempts=5` |
| T7 | **数据零丢失** | 三个 app 在 1~2 个周期内全部拉到指标 |

## 七、断言清单

| # | 断言 | 验证方法 | 通过条件 |
|---|---|---|---|
| T1 | 限流被触发 | 看到 2 条/周期的 `拉取被限流` | ✅ |
| T2 | 心跳不丢 | 每条限流日志前都有 `发送心跳` | ✅ |
| T3 | 节流的进入重投队列 | `入重投队列` 计数 ≈ `拉取被限流` 计数 | ✅ |
| T4 | 重投在 1 个周期内完成 | 同一周期内 `重投执行` 数 == 节流数 | ✅ |
| T5 | 退避生效 | backoffMs 大致符合 500/1s/2s/4s/8s 序列 | ✅ |
| T6 | 5 次上限生效 | 长时挤压可看到 `重投耗尽` | ✅(可选) |
| T7 | **数据零丢失** | `monitor-metrics` Kafka topic(或 InfluxDB)三条 app 的数据都有 | ✅ |

## 八、可能踩坑

### 8.1 host 提取不一致

如果用 `http://localhost:18081` 替代 `http://host.docker.internal:18081`,从 spring-watch 进程看 `host=localhost`;从容器内看 `host=host.docker.internal`,**两者不在同一 bucket,不会触发节流**。

→ 三个 app 的 endpoint **必须**都用 `host.docker.internal`。

### 8.2 mock-test-1 启动慢

`/api/ping` 健康检查 30s 内未必绿,头 5s 的拉取可能全部 `unreachable` → `markInactive`,**看不到节流**(因为根本没拉到 permit 阶段)。

→ 先 `curl http://localhost:19464/metrics` 确认 mock-test-1 真的就绪再注册 app。

### 8.3 数据库残留

如果之前测试有同名 `throttle-test-a/b/c` 残留,register 会失败。

```bash
# 清理
curl -X DELETE http://localhost:8080/api/apps/{id}
# 或
psql -h localhost -U root -d spring_collector \
  -c "DELETE FROM monitor_app WHERE app_name LIKE 'throttle-test-%';"
```

### 8.4 perHostConcurrent 改完没生效

Spring Boot `@ConfigurationProperties` 在启动时一次性绑定;改 `application.yml` 后必须重启 spring-watch。

### 8.5 scrapeInterval 太短

设 1~2s 可能让虚拟线程池过载(`poolSize=32` 够用,但每次拉取也是虚拟线程跑,叠加会爆)。

→ 建议 5s 起,视机器配置调。

## 十一、断言脚本(可选,用 PowerShell 一次性检查)

```powershell
# 假设日志在 spring-watch.log
$log = Get-Content spring-watch.log

$throttled = ($log | Select-String "拉取被限流").Count
$enqueued  = ($log | Select-String "入重投队列").Count
$retried   = ($log | Select-String "重投执行").Count
$exhausted = ($log | Select-String "重投耗尽").Count
$hbBeforeThrottle = ($log | Select-String -Pattern "发送心跳.*\r?\n.*拉取被限流").Count

Write-Host "Throttle-triggered: $throttled"
Write-Host "Enqueued:          $enqueued"
Write-Host "Retry-executed:    $retried"
Write-Host "Exhausted:         $exhausted"
Write-Host "HB-before-throttle:$hbBeforeThrottle"

if ($throttled -gt 0 -and $enqueued -eq $throttled -and $retried -ge $throttled -and $hbBeforeThrottle -eq $throttled) {
    Write-Host "[PASS] 限流 + 降级重投 行为符合预期" -ForegroundColor Green
} else {
    Write-Host "[FAIL] 见上述计数差异" -ForegroundColor Red
}
```
