## 3. зҠ¶жҖҒжңәиҜҰи§Ј(еҖҹйүҙ HertzBeat PeriodicAlertRuleScheduler)

```mermaid
stateDiagram-v2
    [*] --> IDLE: йҰ–ж¬ЎеҗҜеҠЁ / Redis TTL иҝҮжңҹ
    IDLE --> PENDING: breached=true<br/>(йҰ–ж¬Ў)
    PENDING --> PENDING: breached=true<br/>иҮӘеўһ triggerCount
    PENDING --> FIRING: times иҫҫ & durationSeconds еҲ°
    PENDING --> IDLE: breached=false<br/>(дёӯйҖ”жҒўеӨҚ,clear зҠ¶жҖҒ)
    PENDING --> FIRING: PendingStateScanner е…ңеә•<br/>(ж— ж–°дәӢд»¶д№ҹиғҪеҚҮзә§)
    FIRING --> FIRING: breached=true<br/>(еҺ»йҮҚ,дёҚеҸ‘йӮ®д»¶)
    FIRING --> RESOLVED: breached=false<br/>(еҸ‘жҒўеӨҚйӮ®д»¶)
    RESOLVED --> PENDING: breached=true<br/>(йҮҚж–°иҝӣе…ҘзҙҜи®Ў)
    RESOLVED --> IDLE: breached=false<br/>(clear зҠ¶жҖҒ)
    RESOLVED --> [*]: clear
    FIRING --> [*]: clear
```

**и§ҰеҸ‘жқЎд»¶з»ҶеҲҷ:**

| еҪ“еүҚжҖҒ | жқЎд»¶ | еҠЁдҪң |
|---|---|---|
| IDLE | breached | еҶҷ PENDING,и®° firstBreachAt,жё… triggerCount |
| IDLE | жңӘ breached | дҝқжҢҒ IDLE |
| PENDING | breached | иҮӘеўһ triggerCount;иӢҘ `count >= times` дё” `elapsed >= durationSeconds Г— 1000ms` вҶ’ FIRING + `fire()` |
| PENDING | жңӘ breached | `stateStore.clear()` еӣһ IDLE |
| FIRING | breached | и·іиҝҮ(еҺ»йҮҚ,дёҚйҮҚеҸ‘йӮ®д»¶) |
| FIRING | жңӘ breached | еҶҷ RESOLVED + `resolve()` + еҸ‘жҒўеӨҚйӮ®д»¶ + clear |
| RESOLVED | breached | йҮҚж–°иҝӣе…Ҙ PENDING,жё… triggerCount |
| RESOLVED | жңӘ breached | clear |

> **PENDING е…ңеә•и®ҫи®Ў**:иӢҘ `times > 1`,дәӢд»¶жөҒзЁҖз–Ҹж—¶(дҫӢеҰӮ 10s жүҚжқҘдёҖдёӘдәӢд»¶)еҸҜиғҪж°ёиҝңеҮ‘дёҚеӨҹ `times`,жүҖд»Ҙ `PendingStateScanner` жҜҸ 5s жү«дёҖж¬Ў,еҸӘиҰҒ durationSeconds еҲ°дәҶдё” triggerCount зҙҜи®Ўж•°еӨҹдәҶе°ұејәеҲ¶еҚҮзә§еҲ° FIRINGгҖӮ

```mermaid

    participant TGT as зӣ®ж Үеә”з”Ё

    participant KP as KafkaProducerBridge
```
