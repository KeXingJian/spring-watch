# spring-watch 架构图

> 基于当前代码(`src/main/java` 18 个包、`docker-compose.yml`、`application.yml`、`白皮书.md`)绘制的运行时架构。
> 拉取模型:`spring-watch ──HTTP GET──> 目标应用`;目标不主动推送,Java Agent 暴露指标/日志端点。

## 2. 采集数据流(指标 + 日志 + 心跳)

```mermaid
sequenceDiagram
    autonumber
    participant R as CollectScheduleRegistry
    participant T as AppPullTask
    participant H as HostThrottler
    participant M as AgentMetricsCollector
    participant L as AgentLogCollector
    participant P as KafkaProducerBridge
    participant F as KafkaFallbackQueue
    participant K as Kafka
    participant BC as BatchMetricConsumer
    participant BL as BatchLogConsumer
    participant BH as BatchHeartbeatConsumer
    participant I as InfluxDB
    participant PGD as PostgreSQL
    participant RD as Redis

    R->>T: scheduleWithFixedDelay / cronLoop
    T->>T: 可达性探测 http://host:port/metrics
    alt 不可达
        T->>PGD: MonitorApp.status = INACTIVE
    else 可达
        T->>P: sendHeartbeat(appid, host, now)
        T->>H: tryAcquire(host)
        alt 被限流
            T->>F: PullRetryQueue.enqueue
        else 通过
            T->>M: collect(target)
            M-->>P: sendMetric(MetricEvent{appid, metric, value, ts, tags})
            T->>L: collect(appid, endpoint, since)
            L-->>P: sendLog(LogEvent × N)
            T->>PGD: 更新 MonitorApp.lastLogPullTime
        end
    end

    P->>K: send(monitor-metrics / monitor-logs / monitor-heartbeat)
    K-->>BC: batch poll
    K-->>BL: batch poll
    K-->>BH: batch poll
    BC->>I: writePoints(measurement=springboot_metrics)
    BL->>BL: parser → sanitizer → fingerprinter → dedup(RD)
    BL->>I: writePoints(measurement=app_log)
    BH->>PGD: update lastHeartbeat + status=active
```

## 3. 告警评估流(指标 + 日志 + 错误率定时)

```mermaid
flowchart LR
    subgraph M_TRIG["指标告警路径"]
        K1["Kafka: monitor-metrics"]
        BAC["BatchAlertConsumer"]
        AEX1["AsyncAlertExecutor<br/>(虚拟线程)"]
        AE1["AlertEngine.process(MetricEvent)"]
        K1 --> BAC --> AEX1 --> AE1
    end

    subgraph L_TRIG["日志告警路径"]
        BL["BatchLogConsumer"]
        AEX2["AsyncAlertExecutor"]
        AE2["AlertEngine.process(LogEvent)"]
        BL -- "log_keyword / log_new_pattern" --> AEX2 --> AE2
    end

    subgraph R_TRIG["错误率定时任务"]
        LAS["LogAlertScheduler<br/>@Scheduled 1min"]
        LAG["LogAggregator.errorRate<br/>(Flux → InfluxDB)"]
        SE["合成 MetricEvent<br/>metricName=log_error_rate"]
        AEX3["AsyncAlertExecutor"]
        LAS --> LAG --> SE --> AEX3 --> AE3["AlertEngine.process"]
    end

    AE1 & AE2 & AE3 --> EV["AlertEvaluator<br/>JEXL 表达式判断 breached"]
    EV --> ST["AlertStateStore<br/>IDLE→PENDING→FIRING→RESOLVED"]
    ST --> AE["AlertEngine"]
    AE --> RC["AlertRuleCache<br/>(30s 从 PG 刷新)"]
    AE --> PG[("PostgreSQL<br/>alert_history")]
    AE --> NT["AlertNotifier<br/>SMTP 邮件"]
```

**状态机(IDLE → PENDING → FIRING → RESOLVED):**

| 当前状态 | 条件 breached | 条件未 breached |
|---|---|---|
| IDLE | 进入 PENDING,记录 firstBreachAt | 保持 IDLE |
| PENDING | `times` 触发 或 `durationSeconds` 到期 → FIRING + 通知 | 恢复,清状态 |
| FIRING | 持续触发,跳过(去重) | 进入 RESOLVED,关闭 history,发恢复通知 |
| RESOLVED | 重新进入 PENDING | 清状态 |

## 8. 接入端到端时序(注册 → 采集 → 消费 → 告警)

```mermaid
sequenceDiagram
    autonumber
    participant OP as 运维/SRE
    participant API as MonitorAppController(待实现)
    participant SVC as MonitorAppService
    participant PG as PostgreSQL
    participant REG as CollectScheduleRegistry
    participant TGT as 目标 Spring Boot<br/>(Agent + @SpringWatch)
    participant TASK as AppPullTask
    participant K as Kafka
    participant CON as Batch*Consumer
    participant I as InfluxDB
    participant AE as AlertEngine
    participant MAIL as 邮件

    OP->>API: POST /api/apps {appName, endpoint, metricsPort}
    API->>SVC: register
    SVC->>SVC: SnowFlakeIdGenerator → appid(53bit)
    SVC->>PG: insert monitor_app
    SVC->>REG: upsert(MonitorApp)
    REG-->>REG: AppSchedule.from → 注册 INTERVAL/CRON 任务

    loop 每 15s(或 cron)
        REG->>TASK: safeRun(appid)
        TASK->>TGT: HTTP GET /metrics
        TASK->>TGT: HTTP GET /api/agent/logs?since=
        TASK->>K: send(MetricEvent / LogEvent / HeartbeatEvent)
        K-->>CON: batch
        CON->>I: writePoints(springboot_metrics / app_log)
        CON->>AE: process(MetricEvent / LogEvent)
        AE->>PG: alert_history
        AE->>MAIL: firing/resolved
    end
```

> **设计原则回顾(摘自 `白皮书.md`):**
> - 拉取模型:目标**不主动**推送,spring-watch 主动 HTTP GET
> - 平台形态:多应用配置驱动,非单点工具
> - 仅限 Spring Boot 目标应用
> - 指标源头 = Java Agent(不依赖 Actuator / Micrometer)
> - 方法级监控 = `@SpringWatch` 注解驱动
