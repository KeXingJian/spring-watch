# spring-watch 视图层规划

> 静态 HTML + 原生 JS + ECharts CDN,无构建。后端接口需要时再补,本文只规划视图。

---

## 1. 整体策略

### 1.1 选型

| 维度 | 选择 | 理由 |
|---|---|---|
| 技术栈 | HTML5 + 原生 JS(无 Vue/React) | 静态资源,无构建链,文件即用 |
| 图表 | ECharts(CDN) | 监控大屏标配,折线/柱状/饼图/热力图齐全 |
| HTTP | `fetch` 封装 | 浏览器原生 |
| 路由 | iframe + 左侧导航 | 不用 SPA 路由,部署最简 |
| 样式 | 单一 `style.css` | 避免散落 |
| 鉴权 | 暂不接(后续可加 cookie/token) | v1 阶段以功能为主 |

### 1.2 部署位置

```
src/main/resources/static/         # Spring Boot 静态资源默认目录
```

- 浏览器访问 `http://localhost:8080/` 直接打开 `index.html`
- 公共静态资源由 Spring Boot 自动托管,零配置
- CORS 不需要(同源)

### 1.3 布局基线

```
┌──────────────────────────────────────────────────────┐
│  TopBar  spring-watch 平台          [应用切换▾]   [时间] │
├────────┬─────────────────────────────────────────────┤
│        │                                             │
│ Side   │              iframe 区域                    │
│ Nav    │         (各业务页面嵌入)                     │
│        │                                             │
│        │                                             │
└────────┴─────────────────────────────────────────────┘
```

- `index.html` 是壳,左侧导航 + 右侧 `<iframe name="content">`
- 点击菜单项 → `target="content"` 切 iframe,不刷新整页
- 跨页跳转带上下文用 `?appid=xxx` 传参,iframe 内 `location.search` 读取

## 3. 各页面详细设计

### 3.1 控制台首页 `index.html`

**职责**:壳页面,提供布局 + 左侧导航 + 应用切换器。

**结构**:

```html
<header>
  <h1>spring-watch</h1>
  <select id="app-switcher"><!-- 动态加载应用列表 --></select>
  <span class="clock"></span>
</header>
<aside>
  <nav>
    <a target="content" href="pages/dashboard.html">总览</a>
    <a target="content" href="pages/apps.html">监控应用</a>
    <a target="content" href="pages/app-detail.html">应用指标</a>
    <a target="content" href="pages/logs.html">日志分析</a>
    <a target="content" href="pages/alert-rules.html">告警规则</a>
    <a target="content" href="pages/alert-history.html">告警历史</a>
    <a target="content" href="pages/email-config.html">邮箱配置</a>
  </nav>
</aside>
<main>
  <iframe name="content" src="pages/dashboard.html"></iframe>
</main>
```

**应用切换器**:
- 调 `/api/apps` 列出所有 active 应用
- 选中后写入 `localStorage.currentAppid`
- 各业务页启动时读 `localStorage.currentAppid` 作为默认 `appid`
- 提供"无选中"占位:某些页面(规则、邮箱配置、监控应用管理)与 `appid` 无关,可不选

### 3.2 总览 Dashboard `pages/dashboard.html`

**职责**:平台级总览,一眼看出系统健康度。

**布局**:6 卡片栅格

| 区域 | 内容 | 数据源 |
|---|---|---|
| 应用总数 | active / paused / total | `GET /api/apps` |
| 采集健康 | 最近 1h 内有心跳的应用数 / 总数 | `monitor_app.lastHeartbeat` |
| 待处理告警 | PENDING / FIRING 数量 | `alert_history.status` |
| 24h 告警趋势 | 折线图(每小时触发数) | InfluxDB / 聚合接口 |
| Top 异常模式 | Top 10 错误日志 pattern | `GET /api/logs/patterns` |
| 采集耗时 | 各应用最近一次拉取耗时柱状图 | `Micrometer` 自监控 |

**无强依赖** `appid`(平台级),应用切换器置灰。

### 3.3 监控应用-列表 `pages/apps.html`

**职责**:`monitor_app` 增删改查。

**表格列**:

| appid | appName | endpoint | metricsPort | scheduleType | cron | scrapeInterval | status | lastHeartbeat | 操作 |
|---|---|---|---|---|---|---|---|---|---|
| 雪花 | 名称 | host:port | 9464 | INTERVAL/CRON | cron 表达式 | 15s | active/paused | 时间 | 编辑/暂停/恢复/删除/查看指标 |

**顶部工具栏**:
- `[+ 新增应用]` 按钮 → 弹窗加载 `app-form.html`
- 状态过滤(全部 / active / paused)
- 关键字搜索(appName)

**操作列按钮**:
- **编辑** → 弹窗表单(同新增)
- **暂停/恢复** → 状态切换,带二次确认
- **删除** → 二次确认(提示"会同时取消调度")
- **查看指标** → 跳转 `app-detail.html?appid=xxx`
- **生成 OTel 配置** → 弹窗显示接入命令(环境变量 + `-javaagent`)

**表头排序**:appid / appName / lastHeartbeat 可点击排序。

### 3.4 监控应用-表单 `pages/app-form.html`

**职责**:新增 / 编辑共用,通过 URL `?id=xxx` 区分。

**字段** (对应 `AppRegisterRequest` + `MonitorApp`):

| 字段 | 控件 | 必填 | 说明 |
|---|---|---|---|
| appName | input text | ✅ | 唯一,前端查重提示 |
| endpoint | input text | ❌ | host:port 或域名,留空允许后续编辑 |
| metricsPort | input number | ❌ | 默认 9464 |
| appType | select | ❌ | 固定 springboot(v1) |
| scheduleType | select | ✅ | INTERVAL / CRON |
| scrapeInterval | input number | ✅ | INTERVAL 时用,默认 15(秒) |
| cronExpression | input text | ❌ | CRON 时用,7 段校验 |
| labels | input text | ❌ | key=value,key=value 格式 |
| status | select | ✅ | active / paused(编辑时) |

**前端校验**:
- `appName` 非空 + 长度 ≤ 128
- `endpoint` 形如 `host:port`(简单正则)
- `cronExpression` 7 段且每段在合法范围
- `scheduleType=CRON` 时 `cronExpression` 必填

**提交**:
- 新增: `POST /api/apps`(用 `AppRegisterRequest`)
- 编辑: `PUT /api/apps/{id}`(需后端补 Controller)
- 成功后弹 Toast + 通知父页刷新 + 关闭弹窗

### 3.5 应用详情-指标 `pages/app-detail.html`

**职责**:单一应用指标总览,6 大区域 tab 切换。

**URL**:`?appid=xxx` 必传。

**顶部应用信息条**:
```
appName | appid=12345 | endpoint=10.0.0.1:8080 | status=active | lastHeartbeat=2s ago
```

**Tab 结构**:

| Tab | 区域 | 数据源 |
|---|---|---|
| 总览 | 关键指标卡片 + 健康度评分 | 多接口聚合 |
| 系统 | CPU / 内存 / 磁盘 / 网络 / 负载 | InfluxDB `system_*` |
| HTTP | 请求量 / 响应时间 / 状态码分布 / 慢请求 TopN | InfluxDB `http_*` |
| 方法 | `@SpringWatch` 方法调用次数 / P50/P95/P99 | InfluxDB `springboot_metrics(method=)` |
| 日志 | 错误率 / Top 模式 / 突增 | `GET /api/logs/stats/error-rate` `GET /api/logs/patterns` `GET /api/logs/anomaly` |
| JDBC | 连接池活跃 / 慢 SQL | InfluxDB `jdbc_*` |
| JVM | 堆 / 非堆 / GC / 线程 / 类加载 | InfluxDB `jvm_*` |

**交互**:
- 顶部时间选择器:近 5min / 15min / 1h / 6h / 24h / 自定义
- 自动刷新开关(默认 30s,可关)
- 每个图表支持"放大查看"→ 弹窗大图
- 指标卡片下钻:点击跳到对应的明细页(方法 Tab → 方法调用明细表)

**共享子组件**:
- 时间选择器(各 tab 通用,放最顶)
- 折线图 wrapper(传 data/options 即可)
- 数值卡片(标题 / 当前值 / 同比箭头 / 阈值线)

### 3.6 日志分析 `pages/logs.html`

**职责**:日志检索 + 上下文 + 统计 + 异常模式。

**URL**:`?appid=xxx` 必传。

**左侧筛选栏**(常驻):

| 控件 | 说明 |
|---|---|
| 应用选择器 | 必选,默认取 `localStorage.currentAppid` |
| 时间范围 | 默认近 15min,快捷:5m / 15m / 1h / 6h / 24h / 自定义 |
| 日志级别 | 多选:DEBUG/INFO/WARN/ERROR |
| 关键字 | 全文检索(message / throwable) |
| logger | 下拉自动补全(基于已有数据) |
| threadName | 输入 |
| 按钮 | [搜索] [重置] [保存视图] |

**主区 Tabs**:

| Tab | 内容 | 数据源 |
|---|---|---|
| 列表 | 分页表格,行点击展开完整 message + 异常栈 | `GET /api/logs/search` |
| 错误率 | 折线图(total / error / warn) | `GET /api/logs/stats/error-rate-series` |
| Top 模式 | 表格:fingerprint / 出现次数 / 最后出现 / 样本 | `GET /api/logs/patterns` |
| 异常突增 | 卡片:当前 / 上窗口 / 倍数 / 阈值 | `GET /api/logs/anomaly` |
| Trace | 输入 traceId 拉取串联日志 | `GET /api/logs/trace/{traceId}` |
| 上下文 | 列表中"查看上下文"→ 弹窗 | `GET /api/logs/context` |
| 去重 Top | 高频去重模式(噪声) | `GET /api/logs/dedup/top` |

**列表行**:
```
时间 | 级别 | logger | thread | message(可展开) | [上下文] [复制] [按指纹查]
```

**上下文弹窗**:基于选中行的时间戳 + threadName/host/logger 拉 ±30s 日志,按时间正序展示。

### 3.8 告警规则-表单 `pages/alert-rule-form.html`

**职责**:新增 / 编辑告警规则。

**字段**:

| 字段 | 控件 | 必填 | 说明 |
|---|---|---|---|
| 关联应用 | select(来自 /api/apps) | ✅ | 取 `app.appid` |
| 规则名 | input text | ✅ | 长度 ≤ 256 |
| 规则类型 | radio | ✅ | metric / log_error_rate / log_keyword / log_new_pattern |
| 表达式 | textarea | 条件 | metric: JEXL;log_keyword: 关键字;log_new_pattern: 留空 |
| 阈值 | input number | 条件 | log_error_rate: 百分比 0~100 |
| 持续时长 | input number | ❌ | durationSeconds,默认 60 |
| 连续次数 | input number | ❌ | times,默认 1 |
| 告警级别 | select | ✅ | INFO / WARNING / CRITICAL,默认 WARNING |
| 通知渠道 | textarea(JSON) | ❌ | `{"email":"a@x.com,b@y.com"}` |
| 模板 | textarea | ❌ | 占位符提示 |
| 状态 | select | ✅ | enabled / disabled |

**联动**:
- `ruleType=metric` → 表达式必填,阈值隐藏
- `ruleType=log_error_rate` → 阈值必填,表达式隐藏
- `ruleType=log_keyword` → 表达式当关键字,阈值/持续时长/次数隐藏
- `ruleType=log_new_pattern` → 表达式/阈值/持续时长/次数均隐藏

**JEXL 编辑器**(v1 简化):
- 普通 textarea + 占位符提示(变量:metric / value / tags)
- 后续可升级为 monaco-editor(CDN)

**校验**:
- JEXL 提交前用 `new JexlEngine().createExpression(expr)` 前端预解析,失败提示
- JSON 通知渠道 `JSON.parse` 校验

## 4. 公共组件

### 4.1 全局 JS 模块 `js/common.js`

封装以下能力,所有页面 `<script src="/js/common.js">` 引用:

```js
window.SW = {
  // API 客户端
  api: {
    get(url, params)         // → Promise<data>
    post(url, body)
    put(url, body)
    del(url)
  },
  // 应用上下文
  ctx: {
    getAppid()               // 读 localStorage.currentAppid
    setAppid(appid)          // 写 localStorage + 通知其他 tab
    listApps()               // → Promise<List<MonitorApp>>
  },
  // UI
  ui: {
    toast(msg, type)         // type: info/success/warn/error
    confirm(msg)             // → Promise<boolean>
    modal(opts)              // 打开弹窗(html 字符串 / url)
    closeModal()
  },
  // 工具
  util: {
    fmtTime(iso)
    fmtDuration(ms)
    escapeHtml(str)
    copyToClipboard(text)
  },
  // 图表 wrapper(详见 §4.4)
  chart: {
    line(el, data, opts)         // 折线/面积,data = [{name, points:[[t,v],...]}]
    bar(el, data, opts)          // 柱状,data = {categories:[], series:[{name,data:[]}]}
    pie(el, data, opts)          // 饼图,data = [{name, value}]
    gauge(el, value, opts)       // 仪表盘,value = 当前数值
    heatmap(el, data, opts)      // 热力图(方法调用矩阵)
    scatter(el, data, opts)      // 散点图(响应时间-调用次数)
    numberCard(el, value, opts)  // 数值卡片(带同比/阈值线)
    destroy(el)                  // 释放实例
  }
}
```

> ECharts 通过 CDN 引入:`<script src="https://cdn.jsdelivr.net/npm/echarts@5.5.0/dist/echarts.min.js"></script>`
> 不引入主题文件,统一在 `SW.chart` 内置暗色/亮色变量,默认走亮色。

### 4.4 ECharts 图表清单与封装

> **CDN**:`echarts@5.5.0`,全部页面通过 `SW.chart.xxx(el, data, opts)` 统一调用,不直接 `echarts.init`。
> 颜色变量取自 `style.css` 的 CSS 变量,`SW.chart` 内部读 `:root` 的 `--c-primary` 等。

#### 4.4.1 各页面图表对照表

| 页面 | 图表 | 类型 | 数据源 | 调用 |
|---|---|---|---|---|
| **总览 dashboard** | 应用状态饼图 | `pie` | `monitor_app.status` 聚合 | `SW.chart.pie` |

|  | 健康度评分 | `gauge` | 综合计算 | `SW.chart.gauge` |
| **应用详情-系统** | CPU 使用率 | `line(area)` | `system_cpu_usage` | `SW.chart.line` |
|  | 内存使用 | `line(area)` | `system_mem_*` | `SW.chart.line` |
|  | 磁盘使用 | `gauge` ×N | `system_disk_usage` | `SW.chart.gauge` |

#### 4.4.2 `SW.chart.line` 封装示例

```js
// data 格式:支持多 series
// [
//   { name: 'P50',  points: [['2026-06-17T10:00Z', 12.3], ...] },
//   { name: 'P95',  points: [...] },
//   { name: 'P99',  points: [...] }
// ]
SW.chart.line('#chart-http-rt', data, {
  title: 'HTTP 响应时间',
  yAxisName: 'ms',
  smooth: true,             // 曲线平滑
  area: false,              // 是否面积图
  dualAxis: false,          // 双 y 轴(网络上下行等场景)
  stack: false,             // 堆叠(JDBC 连接池等场景)
  threshold: 500,           // 阈值线(虚线 + 标注)
  timeRange: 'last1h',      // 时间格式:由 SW 内部统一转 ISO
  height: 280
});
```

**内部实现要点**:
- 自动转 `series[].type = 'line'`
- 时间轴用 `xAxis.type = 'time'`
- 颜色按 series 顺序从 `['--c-primary','--c-warn','--c-danger','--c-success','--c-info']` 取
- `tooltip.trigger = 'axis'`,带 crosshair
- `dataZoom` 内置(默认滑动条 + inside 缩放)
- 响应式:`window.resize` 自动 `chart.resize()`
- 实例挂到 `el.__echarts__`,便于 `SW.chart.destroy(el)`

## 5. 路由与上下文传递

### 5.1 应用选择器

- 全局唯一一个,放 `index.html` 顶栏
- 选择后写 `localStorage.currentAppid`
- 各业务页通过 `SW.ctx.getAppid()` 获取
- 业务页打开时如果未选 → 弹窗提示"请先选择应用"或自动跳到应用列表

### 5.2 跨页跳转

- iframe 内跳到另一页:用 `parent.SW.ctx.setAppid(appid)` 写入,再 `parent.document.querySelector('iframe').src = 'xxx'`
- 业务页跳转:`top.location.hash` 不适合(用了 iframe),改用 `parent.window.SW.nav.go(url)` 统一入口

### 5.3 URL 参数

- 所有"应用相关"页面都用 `?appid=xxx`,优先级高于 `localStorage`
- 时间范围统一用 `?from=ISO&to=ISO`,可分享链接

## 9. 反模式(防止返工)

- ❌ **禁止**引入 Vue/React/npm/webpack — 与"无构建"原则冲突
- ❌ **禁止**把多个业务页塞进一个 `index.html` 用 hash 切换 — iframe 隔离更清晰
- ❌ **禁止**在页面里写原生 `fetch` 不走 `SW.api` — 后续换 token/拦截器要全改
- ❌ **禁止**把 ECharts option 散落在各页面 — 统一从 `SW.chart` 出
- ❌ **禁止**时间选择器各页面自己实现 — 抽公共组件
- ❌ **禁止**用 localStorage 存敏感信息(密码 / token)— v1 暂不存,后续 cookie 化
- ❌ **禁止**跨域调后端 — 同源部署,无 CORS 问题
- ❌ **禁止**用 jQuery — 原生 JS + ECharts 已够
