## 一、改造目标与原则

### 1.1 三大改造目标

| 目标 | 现状 | 目标 | 优先级 |
|---|---|---|---|
| **用户能选告警等级** | 引擎硬编码 warning/critical | 规则表里自定义 | P0 |
| **支持多条件表达式** | 只能写 `metricName > value` 单条 | 能写 `a > 1 && b < 2` | P1 |
| **触发次数 + 消息模板** | 只有 `duration` | `times` 防抖动 + `{{value}}` 占位符 | P2 |

### 1.2 五条核心原则

1. **向后兼容**:旧规则在升级后行为不变
2. **独立发布**:每阶段 P0/P1/P2 可独立 ship,不依赖后续阶段
3. **可回滚**:每阶段配对应的回滚 migration
4. **单测先行**:每阶段必须配单测,覆盖率 ≥ 80%
5. **零停机**:升级过程中已发到 Kafka 的事件不丢,状态机不破坏

private String buildSubject(AlertRule rule, MetricEvent event, String type) {
    String appName = rule.getApp() != null ? rule.getApp().getAppName() : "appid=" + event.getAppid();
    String level = determineLevel(rule);
    String prefix = "firing".equals(type)
            ? "[" + level.toUpperCase() + "]"
            : "[RESOLVED]";
    return String.format("%s %s - %s = %.2f (rule: %s)",
            prefix, appName, event.getMetricName(),
            event.getValue() != null ? event.getValue() : 0.0,
            rule.getRuleName() != null ? rule.getRuleName() : "rule-" + rule.getId());
}

private String determineLevel(AlertRule rule) {
    if (rule.getLevel() == null || rule.getLevel().isBlank()) {
        return "WARNING";
    }
    return rule.getLevel().toUpperCase();
}

### 4.4 JexlExprEvaluator(核心)

```java
// alerter/JexlExprEvaluator.java
@Slf4j
@Component
@RequiredArgsConstructor
public class JexlExprEvaluator {

    private final JexlEngine jexlEngine;

    public boolean evaluate(String expression, MetricEvent event) {
        if (expression == null || expression.isBlank()) {
            return false;
        }
        try {
            JexlExpression expr = jexlEngine.createExpression(expression);
            JexlContext ctx = new MapContext();
            ctx.set("value", event.getValue());
            ctx.set("metric", event.getMetricName());
            ctx.set("__app__", String.valueOf(event.getAppid()));
            ctx.set("__count__", event.getCount());
            // 扩展点:把 tags 平铺到顶层
            if (event.getTags() != null) {
                event.getTags().forEach(ctx::set);
            }
            Object result = expr.evaluate(ctx);
            return Boolean.TRUE.equals(result);
        } catch (JexlException e) {
            log.warn("[Alerter] JEXL 表达式执行失败 - expr={}, error={}", expression, e.getMessage());
            return false;
        } catch (Exception e) {
            log.warn("[Alerter] JEXL 表达式执行异常 - expr={}, error={}", expression, e.getMessage(), e);
            return false;
        }
    }
}
```

### 4.5 AlertEvaluator 改造(自动检测格式)

```java
// alerter/AlertEvaluator.java —— 关键改动
@Slf4j
@Component
@RequiredArgsConstructor
public class AlertEvaluator {

    // 简单格式: metricName op value
    private static final Pattern SIMPLE_EXPR =
            Pattern.compile("^(\\w[\\w.]*)\\s*(>=|<=|>|<|==|!=)\\s*([\\d.]+)$");

    private final JexlExprEvaluator jexlEvaluator;

    public boolean isBreached(AlertRule rule, MetricEvent event) {
        if (rule == null || event == null) return false;
        if (!"metric".equals(rule.getRuleType())) return false;
        if (rule.getExpression() == null || rule.getExpression().isBlank()) return false;

        String expr = rule.getExpression().trim();

        // 简单格式:走旧逻辑(向后兼容)
        if (SIMPLE_EXPR.matcher(expr).matches()) {
            return simpleEvaluate(expr, event);
        }

        // 复杂格式:走 JEXL
        return jexlEvaluator.evaluate(expr, event);
    }

    private boolean simpleEvaluate(String expr, MetricEvent event) {
        Matcher m = SIMPLE_EXPR.matcher(expr);
        if (!m.matches()) return false;
        String metricName = m.group(1);
        String op = m.group(2);
        double threshold = Double.parseDouble(m.group(3));

        if (event.getMetricName() == null || !metricName.equals(event.getMetricName())) {
            return false;
        }
        Double value = event.getValue();
        if (value == null) return false;

        return switch (op) {
            case ">"  -> value >  threshold;
            case "<"  -> value <  threshold;
            case ">=" -> value >= threshold;
            case "<=" -> value <= threshold;
            case "==" -> value == threshold;
            case "!=" -> value != threshold;
            default   -> false;
        };
    }
}
```

**关键**:在 `AlertEvaluator` 类头上**不能继续用** `@RequiredArgsConstructor` + 单一 `JexlExprEvaluator` 依赖(因为我们重构了构造),改成显式 `@RequiredArgsConstructor` + `private final JexlExprEvaluator jexlEvaluator`。

### 5.5 AlertEngine handleBreach 改造

```java
// AlertEngine.java —— handleBreach 关键段
private void handleBreach(AlertRule rule, MetricEvent event,
                          AlertState current, Instant now) {
    Long ruleId = rule.getId();
    Long appid = event.getAppid();

    if (current == AlertState.FIRING) {
        log.trace("[Alerter] 告警持续中 - ruleId={}, appid={}", ruleId, appid);
        return;
    }

    if (current == AlertState.IDLE || current == AlertState.RESOLVED) {
        stateStore.setState(ruleId, appid, AlertState.PENDING, now, null);
        stateStore.clearTriggerCount(ruleId, appid);  // 新增:重置计数
        log.debug("[Alerter] 条件首次满足, 进入PENDING - ruleId={}, appid={}", ruleId, appid);
        return;
    }

    if (current == AlertState.PENDING) {
        Instant firstBreach = stateStore.getFirstBreachAt(ruleId, appid);
        if (firstBreach == null) {
            stateStore.setState(ruleId, appid, AlertState.PENDING, now, null);
            return;
        }

        int times = rule.getTimes() == null ? 1 : rule.getTimes();
        int durationSeconds = rule.getDurationSeconds() == null ? 60 : rule.getDurationSeconds();

        // 双触发条件:duration 到 OR 触发次数到(取先满足的)
        boolean durationMet = Duration.between(firstBreach, now).toMillis() >= durationSeconds * 1000L;
        if (durationMet || times <= 1) {
            // 简单模式:duration 到就触发
            stateStore.setState(ruleId, appid, AlertState.FIRING, firstBreach, now);
            fire(rule, event, now);
        } else if (times > 1) {
            // 复杂模式:计数触发
            long count = stateStore.incrementTriggerCount(ruleId, appid);
            if (count >= times) {
                stateStore.setState(ruleId, appid, AlertState.FIRING, firstBreach, now);
                fire(rule, event, now);
            } else {
                log.debug("[Alerter] 触发计数中 - ruleId={}, count={}/{}", ruleId, count, times);
            }
        }
    }
}
```

**简化设计**:**不引入 period 窗口**,因为 spring-watch 现有 `duration_seconds` 已经是"持续时长"语义,与 times 互补:
- `duration_seconds=60, times=1` = 持续 60s 触发(现有默认)
- `duration_seconds=0, times=3` = 3 次事件触发(P2 新增,Redis 计数)
- `duration_seconds=60, times=1` = 同 60s 触发(duration 优先)

### 5.6 AlertNotifier template 渲染

```java
// AlertNotifier.java
private String buildBody(AlertRule rule, MetricEvent event, String type) {
    if (rule.getTemplate() != null && !rule.getTemplate().isBlank()) {
        return renderTemplate(rule.getTemplate(), rule, event, type);
    }
    return defaultBody(rule, event, type);  // 原有逻辑
}

private String renderTemplate(String template, AlertRule rule, MetricEvent event, String type) {
    String level = determineLevel(rule);
    String appName = rule.getApp() != null ? rule.getApp().getAppName() : "appid=" + event.getAppid();
    String value = event.getValue() != null ? String.format("%.2f", event.getValue()) : "null";
    String threshold = rule.getThresholdValue() != null ? String.format("%.2f", rule.getThresholdValue()) : "null";
    String time = Instant.now().toString();

    return template
            .replace("{{level}}", level)
            .replace("{{type}}", type.toUpperCase())
            .replace("{{app}}", appName)
            .replace("{{appid}}", String.valueOf(event.getAppid()))
            .replace("{{metric}}", event.getMetricName() != null ? event.getMetricName() : "")
            .replace("{{value}}", value)
            .replace("{{threshold}}", threshold)
            .replace("{{rule}}", rule.getRuleName() != null ? rule.getRuleName() : "")
            .replace("{{time}}", time);
}
```
