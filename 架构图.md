## 2. 数据流时序图（指标采集 → 告警通知）

```mermaid
sequenceDiagram
    autonumber
    participant S as CollectorScheduler
    participant P as PrometheusScraper
    participant T as Spring Boot Target
    participant KP as KafkaProducerBridge
    participant K as Kafka(metrics)
    participant MC as MetricConsumer
    participant IDB as InfluxDB
    participant AC as AlertConsumer
    participant AE as AlertEvaluator
    participant N as AlertNotifier
    participant U as 用户/IM

    loop 每 scrape_interval 秒
        S->>P: 调度抓取
        P->>T: HTTP GET /actuator/prometheus
        T-->>P: 指标文本
        P->>P: 解析为 MetricEvent
        P->>KP: send(MetricEvent)
        KP->>K: produce(topic=metrics)
    end

    K->>MC: 投递 MetricEvent
    MC->>IDB: WritePoint 批量写入

    K->>AC: 投递指标(告警评估)
    AC->>AE: evaluate(rule, metric)
    AE->>AE: AlertWindowManager 持续时长判断
    alt 触发阈值
        AE->>N: fire(alert)
        N->>U: Webhook/企业微信/钉钉
        N->>IDB: 持久化 alert_history
    end
```

## 5. 关键设计要点

| 模块 | 职责 | 关键技术 |
|------|------|----------|
| **web/** | REST API 接入 | Spring MVC, Validation, ApiResponse 统一封装 |
| **service/** | 业务编排、缓存、查询 | Spring Data JPA, Redis Cache, InfluxDB Query |
| **collector/** | 主动采集 + 推送 | `@Scheduled`, WebClient, KafkaTemplate |
| **consumer/** | 异步消费 + 批量落库 | `@KafkaListener`, JPA Batch Insert, Influx WritePoint |
| **alerter/** | 规则匹配、窗口抑制、通知 | 表达式解析, 持续时长窗口, Webhook |
| **config/** | 基础设施装配 | DataSource, Kafka, Redis, InfluxDB, Flyway |
| **model/** | 领域对象 | JPA Entity, Event(DTO), Request/Response DTO |
| **repository/** | 数据访问 | Spring Data JPA Repository |
